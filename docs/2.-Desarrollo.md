comentarios: En esta parte se procede a identificar las entidades necesarias para llevar a cabo el desarrollo del Software permitiendo cumplir con los requisitos Funcionales definidos. Aclarar que por decisión del desarrollador, se definen las entidades, despues las relaciones entre ellas, se hace el Diagrama Entidad-Relación, y una vez teniendolo se procede a normalizar la DataBase.

# Entidades

    *Usuarios       {id, nombre, a_paterno, a_materno, usuario, password, tipo_usuario}
    *Productos      {id, codigo, nombre, descripcion, categoria, unidad_medida, stock, precio, stock_minimo, link_imagen}
    *Proveedores    {id, nombre, telefono, direccion, email} 
    *Clientes       {id, nombre, telefono,}
    *Pedidos        {id, id_cliente, fecha, total estado}
    *Ventas         {id, fecha, total, total} (el id_cliente puede ser NULL para venta en mostrador)
    *Movimientos    {id, id_producto, tipo_movimiento, cantidad, stock_anterior, stock_actual, motivo, fecha, id_usuario, observaciones, precio_unitario, id_referencia}
    



# NOTAS SOBRE LA MARCHA
